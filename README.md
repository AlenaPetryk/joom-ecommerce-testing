# QA Portfolio: E-commerce Testing Project (Joom Web App)

## 📌 Project Overview
This repository contains a comprehensive testing project for an e-commerce platform. The goal was to perform functional, UI/UX, and business logic testing using professional QA methodologies and tools.

## 🛠 Tools & Technologies
*   **Testing Tool:** Chrome DevTools (Elements, Console, Network, Device Mode)
*   **Documentation:** Markdown (MD)
*   **Environment:** Windows 11 (Desktop Chrome) & **iPhone SE Emulation**
*   **Version Control:** Git & GitHub

## 🧪 What's Inside?
I have organized the testing artifacts into a clear structure:

1.  **[Test Execution Report](docs/test-cases/execution-report.md)**: A master list of **33 test cases** covering the entire user journey—from Email Registration and Social OAuth to Checkout and Payment processing.
2.  **[Regression Checklist](docs/checklists/regression-checklist.md)**: A high-level suite for rapid verification of core e-commerce flows.
3.  **[Bug Reports Registry](docs/bug-reports/bug-registry.md)**: Detailed reports for 9 identified defects, including reproduction steps, severity, and visual proofs.
4.  **[Test Design Analysis](docs/test-design/analysis-summary.md)**: Application of **EP**, **BVA**, **Decision Tables**, and **State Transition Testing** to optimize coverage.

## 📈 Key Testing Highlights
*   **Full User Journey:** Tested registration via email (including email confirmation checks) and social media login.
*   **Test Design Mastery:** Applied State Transition Testing for order lifecycles and Decision Tables for complex coupon logic.
*   **Mobile Emulation:** Identified critical UI/UX issues (Layout Shifts, Tap Area violations) specifically on small-screen devices (iPhone SE).
*   **Boundary Testing:** Discovered boundary bypass bugs in price filters through rigorous BVA analysis.
